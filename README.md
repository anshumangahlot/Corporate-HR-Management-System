# Corporate HR Management System

## Overview
This project is a  **Corporate HR Management System database** designed using proper normalization. It stores and manages employee-related data like departments, roles, attendance, payroll, and projects in a structured way.

The goal was to build a clean database with **minimum redundancy and strong relationships between tables**.

---

## What’s Done So Far
- Designed database schema  
- Applied normalization (up to 4NF)  
- Created all tables with primary and foreign keys  
- Added sample data (2 records per table)

---

## Main Tables
- Employee, Department, Branch  
- Job_Role, Projects  
- Attendance_Log, Leave_Request  
- Payroll, Salary_Breakdown  
- Recruitment, Meeting  

Supporting tables:
- Employee_Phones, Role_Skills  
- Branch_Dept, Employee_Projects  
- Intern, Full_Time  
- Zip_Directory  

---

## Key Concepts Used
- **Normalization (1NF → 4NF)**  
- **Primary & Foreign Keys**  
- **Junction tables (many-to-many relationships)**  
- **Specialization (Intern / Full-Time employees)**
